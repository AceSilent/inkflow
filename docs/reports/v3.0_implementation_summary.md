## 📦 交付清单

### 核心数据模型 ✅

**文件**: `src/core/models_v3.py`

1. **CharacterMemory** - 角色独立记忆库
   - `known_facts`: 已知事实
   - `false_beliefs`: 错误认知（戏剧冲突来源）
   - `hidden_motive`: 隐藏动机
   - `public_status`: 表面身份
   - `has_information_gap()`: 检测信息差

2. **EmotionalBeat** - 情绪节拍
   - `score`: 情绪值（0-100）
   - `dominant_emotion`: 主导情绪
   - `key_event`: 关键事件

3. **MacroTensionState** - 宏观张力状态
   - `emotional_curve`: 最近5章情绪值
   - `needs_setback()`: 检测是否需要注入挫折
   - `needs_relief()`: 检测是否需要缓解压力

4. **PlotOption** - 剧情选项
   - `core_concept`: 核心概念
   - `surface_plot`: 常规发展
   - `devil_twist`: 魔鬼反转
   - `dramatic_irony`: 戏剧讽刺

5. **BrainstormResult** - 头脑风暴结果
   - `options`: 3个剧情选项
   - `selected_option_id`: 人类选择
   - `human_notes`: 人类批注

6. **SceneOutlineV3** - 场景细纲 v3.0
   - `information_gaps`: 信息差配置
   - `character_motives`: 角色动机
   - `subtext_guidance`: 潜台词指导

7. **IcebergDraftOutput** - 冰山引擎输出
   - `internal_script`: 内心推演
   - `final_prose`: 最终正文

---

### 创意对抗沙盘 ✅

**文件**: `src/agents/brainstorming.py`

1. **ProposerAgent** - 提案 Agent
   - `generate_proposals()`: 生成3个基础剧情方向
   - 支持不同风格（直接对抗 vs 智取布局 vs 借力打力）

2. **DevilsAdvocateAgent** - 魔鬼代言人 Agent
   - `inject_twists()`: 注入致命反转
   - 使用公式：[原计划] + [信息差] = [灾难性后果]
   - 极其毒舌和刁钻的提示词

3. **BrainstormingRoom** - 创意对抗沙盘
   - `brainstorm()`: 执行完整头脑风暴
   - 整合 Proposer + Devil's Advocate

**核心提示词**（Devil's Advocate）:
```
你是顶级剧情破坏者。你的任务是摧毁平庸。
公式：[角色原计划] + [因为某个他不知道的信息差] = [事与愿违的灾难性后果]
```

---

### 潜台词渲染引擎 ✅

**文件**: `src/agents/iceberg_engine.py`

1. **FinalProseExtractor** - 正文提取器
   - `extract_final_prose()`: 从 `<Final_Prose>` 标签提取
   - `extract_internal_script()`: 从 `<Internal_Script>` 标签提取

2. **IcebergAuthor** - 冰山作者
   - `generate_scene_with_subtext()`: 生成带潜台词的场景
   - 强制 CoT：先生成 `<Internal_Script>`，再生成 `<Final_Prose>`

3. **IcebergEngine** - 冰山引擎
   - `render_scene()`: 渲染场景（返回纯正文）
   - `render_scene_with_debug()`: 渲染场景（含 Internal Script）

**冰山理论铁律**:
```
1. 绝对禁止直白表达意图
2. 言不对心：心里想的 ≠ 嘴上说的
3. 动作泄密：真实情绪通过细微动作泄露
```

**提示词模板**（文档中的标准）:
```
<Internal_Script>
分析：（角色的核心心理博弈点）
角色A（潜台词）：...
角色A（实际台词）：...
</Internal_Script>

<Final_Prose>
（最终正文，严格贯彻潜台词逻辑）
</Final_Prose>
```

---

### 状态突变逻辑 ✅

**文件**: `src/core/state_updater_v3.py`

1. **StateUpdater** - 状态更新器
   - `update_from_scene()`: 从场景文本中提取认知变化
   - 使用轻量级 LLM（GPT-4o-mini / Claude 3 Haiku）
   - 自动更新 known_facts 和 false_beliefs

2. **EmotionalBeatTracker** - 情绪节拍追踪器
   - `analyze_scene_emotion()`: 分析场景情绪指数
   - `needs_setback`: 检测是否需要注入挫折
   - `needs_relief`: 检测是否需要缓解压力

**状态变更流程**:
```
场景文本 → LLM 分析 → 提取认知变化 → 更新角色状态
```

---

### Showrunner 工作流 ✅

**文件**: `src/core/showrunner_workflow.py`

**完整状态机**:
```
INIT → USER_INPUT → BRAINSTORMING → USER_DECISION →
OUTLINING → DRAFTING → REVIEWING → EDITOR_JUDGE →
STATE_UPDATE → COMMITTING → USER_INPUT
```

**核心方法**:
- `initialize()`: 加载角色状态和张力状态
- `brainstorm()`: 执行创意对抗
- `select_option()`: 人类选择路线
- `generate_outline()`: 生成细纲（带信息差）
- `draft_scene()`: 冰山引擎渲染
- `update_states()`: 更新角色认知
- `commit_scene()`: 保存场景

**快捷方法**:
- `full_workflow()`: 一键完成所有步骤

## 🎯 核心创新对比

### v2.1 → v3.0 的质的飞跃

| 维度 | v2.1 | v3.0 | 提升 |
|-----|------|------|------|
| **角色状态** | 共享全知视角 | 独立认知状态（信息差） | 🌟🌟🌟🌟🌟 |
| **剧情生成** | Author 直接生成 | 提案 + 魔鬼反转 + 人类决策 | 🌟🌟🌟🌟🌟 |
| **对话质量** | 直接生成正文 | CoT 潜台词推演 + 正文 | 🌟🌟🌟🌟🌟 |
| **情绪控制** | 手动 | 自动检测 + 强制注入 | 🌟🌟🌟🌟 |
| **人类角色** | 观察者 | Showrunner（总运作人） | 🌟🌟🌟🌟🌟 |

### 关键突破

1. **信息差引擎**：让 AI 能够"算计"和"推演"
2. **创意对抗**：防止 AI 生成平庸剧情
3. **潜台词渲染**：对话质量提升 3 个档次
4. **Showrunner 模式**：人类从"打字员"升级为"决策者"

### 完整使用流程

1. **输入灵感**:
   ```
   "主角去京城查案，发现凶手是恩师"
   ```

2. **AI 提案**（3个选项）:

   ```
   选择 B，但把结局改惨一点
   ```

4. **自动生成**:
   - 场景细纲（含信息差配置）
   - 内心推演（潜台词分析）
   - 最终正文（冰山引擎渲染）
   - 状态更新（角色认知变化）

## 🎓 设计哲学

> "不要在底层代码里写死任何'剧情逻辑'，把所有的推演交给 Prompt 和 Pydantic 的 Schema。你只需要构建这个'信息碰撞的物理引擎'，AI 自己会在里面产生奇妙的化学反应。"

**核心原则**:
1. **绝对结构化**：Pydantic 强制约束
2. **逻辑链推演**：优先于文字生成
3. **信息差驱动**：戏剧冲突的来源
4. **人类决策**：Showrunner 模式
