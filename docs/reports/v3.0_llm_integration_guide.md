# AutoNovel-Studio v3.0 LLM集成与书籍隔离说明

## ✅ 已完成的功能

### 1. 书籍隔离（Book Isolation）

**设计原则**：每本书（book_id）作为独立的任务单位，拥有独立的目录和状态文件。

#### 目录结构
```
test_books_output/
├── test_book_001/              # 第一本书
│   ├── config.json             # 书籍配置
│   ├── brainstorming_results/  # 头脑风暴结果
│   │   └── brainstorm_20250115_123456.json
│   ├── scenes/                 # 生成的场景
│   │   └── scene_20250115_123456.txt
│   └── 01_Global_Settings/
│       └── characters_v3.json  # 角色状态（含信息差）
│
└── test_book_002/              # 第二本书（完全隔离）
    ├── config.json
    └── ...
```

#### 关键实现
- **ShowrunnerWorkflow** 构造函数接受 `book_id` 参数
- **独立的存储路径**：`output_dir / book_id /`
- **独立的状态管理**：每个book_id维护独立的角色状态、张力状态
- **配置隔离**：每本书有独立的 `config.json`（基调、类型、世界观、角色）

### 2. LLM集成（Real LLM Calls）

#### 已修改的Agent（占位符 → 真实调用）

**1. ProposerAgent** (`src/agents/brainstorming.py`)
- `generate_proposals()`: 调用 LLM 生成3个基础提案
- 系统提示：定义提案专家角色
- 解析逻辑：正则提取提案A/B/C
- 降级策略：LLM失败时返回占位符

**2. DevilsAdvocateAgent** (`src/agents/brainstorming.py`)
- `_generate_twist()`: 调用 LLM 生成魔鬼反转
- 系统提示：定义剧情破坏者角色（毒舌、刁钻）
- 解析逻辑：提取魔鬼反转、戏剧讽刺、信息差
- 温度设置：0.9（更高创意性）

**3. IcebergAuthor** (`src/agents/iceberg_engine.py`)
- `_call_llm_for_iceberg_output()`: 调用 LLM 生成潜台词对白
- 系统提示：冰山理论大师
- 模板渲染：使用Jinja2动态填充角色信息
- 提取逻辑：从 `<Internal_Script>` 和 `<Final_Prose>` 标签提取

#### LLM客户端
- **支持OpenAI及兼容API**：`OpenAILLMClient`
- **环境变量配置**：
  - `OPENAI_API_KEY`: API密钥
  - `OPENAI_BASE_URL`: API端点（支持DeepSeek、Kimi等）
  - `OPENAI_MODEL`: 模型名称（默认：gpt-4o-mini）

## 🚀 使用方法

### 1. 配置环境变量

创建 `.env` 文件：
```bash
OPENAI_API_KEY=your_api_key_here
OPENAI_BASE_URL=https://api.openai.com/v1  # 可选，支持兼容端点
OPENAI_MODEL=gpt-4o-mini  # 可选
```

### 2. 运行测试

**测试核心功能（无需API）**：
```bash
python test_v3_0.py
```

**测试LLM集成（需要API）**：
```bash
python test_v3_llm_integration.py
```

**启动Gradio UI**：
```bash
python launch_showrunner.py
```

### 3. 代码示例

```python
import asyncio
from src.core.openai_client import OpenAILLMClient
from src.core.showrunner_workflow import ShowrunnerWorkflow

async def main():
    # 初始化LLM客户端
    llm_client = OpenAILLMClient(
        api_key="your_api_key",
        base_url="https://api.openai.com/v1",
        model="gpt-4o-mini"
    )

    # 初始化工作流（书籍隔离）
    workflow = ShowrunnerWorkflow(
        llm_client=llm_client,
        book_id="my_novel_001",  # 每本书独立的ID
        output_dir="books"
    )

    await workflow.initialize()

    # 头脑风暴
    result = await workflow.brainstorm(
        inspiration="主角发现最信任的人其实是反派"
    )

    # 选择选项
    await workflow.select_option(
        option_id="B",
        human_notes="把结局改惨一点"
    )

    # 生成场景
    outline = await workflow.generate_outline(...)
    final_prose = await workflow.draft_scene(...)

asyncio.run(main())
```

## 🎯 关键特性

### 1. 容错设计
所有LLM调用都有 `try-except` 包装，失败时自动降级到占位符：
```python
try:
    response = await llm_client.generate_text(...)
    return parse(response)
except Exception as e:
    logger.error(f"LLM failed: {e}")
    return fallback_placeholder()
```

### 2. 智能解析
多层解析策略应对LLM输出格式变化：
```python
# 策略1：正则精确匹配
pattern = r'提案([ABC])：\s*\n核心概念：(.+?)\s*\n常规发展：(.+?)'

# 策略2：逐行解析
if len(proposals) < 3:
    # 简化匹配逻辑...
```

### 3. 温度控制
不同任务使用不同temperature：
- **提案生成**：0.8（创意性）
- **魔鬼反转**：0.9（极高创意性）
- **冰山渲染**：0.8（平衡创意和一致性）
