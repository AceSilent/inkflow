# 大纲生成方法论 (Outline Generation)

> 从剧情树的confirmed路径生成可执行的Markdown大纲。大纲是给写作Agent的施工图纸。

---

## 生成流程

1. `read_tree()` 获取当前剧情树
2. 找到目标 arc 节点下的所有 confirmed plot_points
3. 将 plot_points 按因果顺序排列
4. 分组为章节（每章 3-5 个 plot_points）
5. 为每章生成详细大纲
6. 用 `save_draft()` 保存到 `02_Outlines/` 目录

## Markdown 大纲格式规范

### 卷纲格式

```markdown
# 第X卷：[卷名]

## 卷目标
[一句话概括本卷核心矛盾和结局方向]

## 主要角色
- [角色A]：本卷中的角色弧线
- [角色B]：本卷中的角色弧线

## 章节概览
1. 第X章：[章名] — [一句话概括]
2. 第X章：[章名] — [一句话概括]
...

## 伏笔计划
- [伏笔A]：在第X章埋下，第Y章揭示
- [伏笔B]：在第X章埋下，跨卷揭示
```

### 章纲格式

```markdown
# 第X章：[章名]

## 章节目标
[本章要达成的叙事任务：推进什么矛盾、揭示什么信息、完成什么角色转变]

## 场景一：[场景标题]
- **视角**：[POV角色]
- **地点**：[具体地点]
- **情节**：[事件序列，用→连接]
- **因果链**：[刺激→察觉→反应→结果]
- **情绪弧**：[起始情绪 → 变化 → 结束情绪]
- **创作焦点**：[给作者的写作指导，指出要注意什么、禁止什么]
- **目标字数**：~[数字]字

## 场景二：[场景标题]
...

## 章末钩子
[本章最后留给读者的悬念/问题]
```

## 从剧情树到大纲的映射规则

| 树节点类型 | 大纲对应 |
|-----------|---------|
| `arc` | → 卷纲标题 + 卷目标 |
| `plot_point` | → 场景或章节概览条目 |
| `branch_point` | → 章节中的决策时刻，在创作焦点中标注 |
| `convergence` | → 群戏场景，需要特别注意多线汇合 |

## 质量检查清单

- [ ] 每个场景都有明确的POV角色
- [ ] 因果链完整（没有突然冒出来的情节）
- [ ] 情绪弧不是平的（每个场景都有情感变化）
- [ ] 创作焦点包含具体的禁止事项（不只是"写好"）
- [ ] 字数目标合理（场景不超过1200字，不低于400字）
- [ ] 章末有钩子
