# Stage 弧线设计 (Stage Arc Design)

> 每个 stage 都是一个完整的情感体验。有开始、有攀升、有高潮、有余韵。

---

## 起承转合结构

### 起（Opening Hook）— 前10%

- 用**动作或对话**开场，不用环境描写铺垫（除非是 stage 首次切换场景）
- 第一句必须包含**悬念、矛盾或反常**
- 让玩家在前三行内知道：谁、在哪、面对什么问题

### 承（Rising Action）— 10%-60%

- 逐步加深矛盾，信息逐层释放
- 至少一次**小反转**（玩家以为A，实际是B）
- 通过 dialogue + action 推进，减少纯 narration 行
- 角色在这个阶段做出**选择**（被动角色是致命的）

### 转（Climax / Turning Point）— 60%-85%

- 本 stage 最核心的冲突爆发点
- 节奏加速：台词变短、action 密集、dialogue 简短
- 情感到达峰值（紧张、愤怒、感动、震惊——取决于场景类型）
- 角色的选择产生**不可逆的后果**

### 合（Resolution + Hook）— 最后15%

- 处理高潮的直接后果（不要悬空）
- 给玩家一个**微小的情感缓冲**
- 最后一段必须包含**下一 stage 的钩子**或**分支选项**

## 情绪曲线设计

每个 stage 的情绪不是直线上升，而是波浪式推进：

```
情绪强度
  ^
  │        ╱╲
  │   ╱╲  ╱  ╲   ← stage 高潮
  │  ╱  ╲╱    ╲
  │ ╱           ╲ ← 钩子/选项
  │╱              
  └──────────────> stage 进度
  起    承    转  合
```

- 不要从第一句就拉满——留出攀升空间
- "承"阶段的小高峰是必要的，防止玩家中途走神
- 高潮后的下降不能太陡——需要2-3行的减速

## 钩子与分支设计

### Stage 末尾钩子类型

1. **悬念钩**："门被推开，站在门口的人不是他预想的……" → 下一 stage 揭晓
2. **决定钩**：角色做出了一个让玩家担心的决定 → 下一 stage 看后果
3. **反转钩**：最后一句推翻玩家之前的假设 → 下一 stage 看真相
4. **情感钩**：留下一个未解决的情感问题 → 玩家想看角色怎么处理

### 分支选项设计

- 2-4 个选项，每个标签 ≤20 字符
- 每个选项应导向有意义的不同叙事分支或奖励
- 选项之间要有**可感知的差异**——不能只是措辞不同但结果一样
- 分支后通过 convergence stage 合流

### 禁忌

- 不要在 stage 末尾做总结性陈述（"他知道，这一切只是开始"——太俗）
- 不要在每个 stage 都用相同类型的钩子
- 不要用假钩子（提出悬念但下一 stage 完全不管它）

## Stage 长度与节奏

- 标准 stage：10-30 行
- 超过 40 行应考虑拆分为两个 stage
- direction（bgm/sfx/bg）通常只在场景开头或氛围转折时设置
- emotion 标签克制使用，只在情绪明显变化时标注

## 创作节奏

- 一次只写一个 stage。写完后用 save_script(package_id, stage_id, stage_json) 保存。
- 写下一个 stage 前，先 read_file 读取已保存的 YAML 回顾前序 stage，确保衔接和伏笔延续。
- 分支 stage（branch_*）可以并行构思，但仍逐个保存。
- convergence stage 必须在所有分支 stage 写完后再写，确保能合理合流。
