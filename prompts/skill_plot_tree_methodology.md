# 剧情树构建方法论 (Plot Tree Methodology)

> 用树的思维构建和探索剧情。主干是命运，分支是可能性，合流是宿命。

---

## 核心概念

剧情树不是文件夹嵌套，而是**因果网络的可视化**。每个节点回答两个问题：
1. "这件事为什么发生？"（depends_on）
2. "这件事发生后，什么变成了可能？"（enables）

## 构建流程

### Phase 1: 主干搭建

从**最重要的转折点**开始，而不是从头开始线性铺设：

1. 确定故事的**核心矛盾**（主角 vs 什么？）
2. 确定 3-5 个**不可跳过的转折点**
3. 用 `add_plot_node(type='plot_point')` 创建这些关键节点
4. 用 `set_causality` 建立因果链
5. 填充转折点之间的过渡节点

### Phase 2: 分支探索

在关键决策点创建分支，探索"如果不这样呢？"：

1. 在每个 `branch_point` 下创建 2-4 个 `candidate` 子节点
2. 每个 candidate 简要描述：走向、后果、对角色弧线的影响
3. **不需要展开细节**——分支探索的目的是看到可能性，不是写完整大纲

### Phase 3: 选择与修剪

人类和 Agent 共同决定走哪条路：

1. 评估每条路径的**叙事价值**：悬念强度、角色成长空间、情感冲击力
2. `confirm_path` 选中的路径
3. `prune_branch` 放弃的路径（记录 reason，以后可能复用）
4. 被修剪的路径不删除——它们是"未被讲述的故事"，可以启发支线

## 分支策略

### 什么时候该分支？

- 角色面临**真正的选择**（不是只有一个合理选项的假选择）
- 存在**多种玩家期待**（玩家可能期望 A，但走 B 更有趣）
- 故事进入**不确定区域**（作者自己也不确定最佳走向）

### 什么时候不该分支？

- 事件的走向是**因果必然**（前面的铺垫已经锁死了结果）
- 分支只影响**微小细节**而不改变大方向
- 当前分支还没走完，不要过早发散

## 合流（Convergence）设计

合流点是剧情树最有戏剧张力的节点——多条线在此汇合：

1. **主动合流**：角色的行动导致线索汇聚。例：所有势力在宗门大比相遇
2. **被动合流**：外部事件迫使分散的角色重聚。例：灾难来临，所有人必须合作
3. **信息合流**：玩家终于看到分散的线索拼成完整图画

合流前，各分支应该让玩家产生**独立的情感投资**。合流时，这些投资碰撞产生**化学反应**。

## 因果链完整性

### 检查清单

- [ ] 每个 `plot_point` 都有至少一个 `depends_on`（它为什么发生？）
- [ ] 每个 `plot_point` 都有至少一个 `enables`（它导致了什么？）
- [ ] 没有"悬空节点"（既不依赖任何东西，也不导致任何东西）
- [ ] 修改某节点时，检查其 `enables` 链上的所有下游节点是否需要调整

### 因果链断裂的信号

- 角色突然出现在不应该在的地方
- 角色使用了还没获得的能力/信息
- 情节推进没有足够的铺垫（玩家会问"这也太巧了吧"）
